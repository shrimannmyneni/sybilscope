SybilScope Pipeline

AGENTIC SYBIL DETECTION ARCHITECTURE
1
Sample & Identify Suspected Root
Random sample of 20 wallets to find the most common funder (suspected sybil hub)
PythonEtherscan API
INPUT
List of wallet addresses (from cache or user input)
PROCESS
sample_and_find_root()
Random sample 20 addresses, check who funded each, find most common funder
OUTPUT
Suspected root funder address + funder frequency distribution
2
Collect On-Chain Data
JAC Walker traverses graph, fetches tx history for every wallet (cache-first, Etherscan fallback)
JAC WalkerEtherscan API
WALKER
DataCollector traverses Wallet nodes
Populates: tx_count, first_funder, protocols, intervals, label
FEATURES EXTRACTED
first_funder (who funded this wallet)
protocol_interactions (contracts called)
operation_intervals (timing gaps)
tx_count (total transactions)
3
Build & Merge Clusters
Two clustering algorithms run in parallel, then merge results by size (most suspicious first)
PythonNetworkX
FUNDER CLUSTERING
build_funding_clusters()
Group wallets by shared first_funder
Sorted by cluster size (largest = most suspicious)
LOUVAIN COMMUNITY DETECTION
build_louvain_clusters()
Build weighted tx graph, run Louvain algorithm
Finds communities beyond shared-funder patterns
MERGE
merge_clusters()
Deduplicate, sort by size descending
Priority: Louvain > Funder (captures more complex relationships)
4
BFS Classification with Agentic Reasoning
Process clusters most-suspicious-first. For each cluster: compute signals, classify with LLM, override, then agentic re-investigation loop.
JAC WalkerGPT-4o-miniPython
SIGNAL 1 — STRUCTURAL
detect_chain()
Linear chain A→B→C→D detection
70% threshold for chain classification
SIGNAL 2 — EVASION
detect_amount_anomaly()
Identical incoming ETH amounts
Statistically impossible by chance
SIGNAL 3 — STRUCTURAL
build_funding_tree()
Multi-hop funder tracing (up to depth 3)
Catches fund-routing obfuscation
SIGNAL 4 — BEHAVIORAL
detect_tx_count_fingerprint()
Identical tx counts across wallets = bot script
e.g., 80% of cluster has exactly 44 txs
SIGNAL 5 — BEHAVIORAL
compute_timing_stats()
Avg gap, std, min/max between txs
Low std/mean ratio = machine precision
SIGNAL 6 — BEHAVIORAL
compute_behavior_similarity()
Jaccard similarity on protocol interactions
>0.5 = identical protocol usage
LLM CLASSIFICATION (by llm())
All 6 signals passed to llm_classify_cluster_enriched()
Single batch call per cluster (GPT-4o-mini, temp=0.1, JSON mode)
Returns: CLEAN | SUSPICIOUS | LIKELY_SYBIL | CONFIRMED_SYBIL + confidence + evidence
POST-LLM OVERRIDE
post_llm_override()
Rule 1: shared funder + cluster ≥3 → min SUSPICIOUS
Rule 2: tx count matches bot fingerprint → bump risk
Rule 3: tx <5 + shared funder → min SUSPICIOUS
Catches LLM false negatives without extra API calls
AGENTIC LOOP MAX 3 ROUNDS
If verdict confidence < 0.7 and risk ≠ CLEAN:
1. Fetch deeper funding tree (depth 3) for that wallet
2. Re-classify with enriched context
3. Repeat until all verdicts ≥ 0.7 or max rounds hit
This is the "agentic" differentiator: the Walker reasons, decides it needs more data, fetches it, and re-reasons.
EARLY STOPPING
If entire cluster is CLEAN → skip remaining clusters
(They have fewer shared funders = less suspicious)
